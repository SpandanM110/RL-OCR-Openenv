# OCR Table RL Environment

[![openenv](https://img.shields.io/badge/openenv-compatible-blue)](https://github.com/meta-pytorch/OpenEnv)
[![HF Space](https://img.shields.io/badge/HuggingFace-Space-yellow)](https://huggingface.co/spaces/SpandanM110/RL-OCR-Openenv)
[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

An **OpenEnv-compatible Reinforcement Learning environment** for structured document table extraction. Agents learn to extract complex tables from document images into structured **Markdown** and **JSON KPIs** through an iterative multi-step feedback loop.

> **Live Demo:** [https://huggingface.co/spaces/SpandanM110/RL-OCR-Openenv](https://huggingface.co/spaces/SpandanM110/RL-OCR-Openenv)

---

## Why This Matters

Current OCR solutions fail spectacularly on real-world documents:

| Problem | What Breaks |
|---|---|
| Multi-level headers | Merged cells misread as separate rows |
| Noisy scans | Characters misidentified (`$` → `S`, `%` → `°/o`) |
| Rotated documents | Row alignment completely lost |
| Semantic extraction | Values extracted but not labeled (what IS this number?) |

**Our approach:** Instead of one-shot OCR, we use **Reinforcement Learning** — the agent extracts, gets scored, and iteratively refines. Crop a region, retry, correct individual cells, and finalize when confident. This mirrors how a human analyst reviews and fixes OCR output.

---

## Novel Contributions

| # | Feature | Description |
|---|---|---|
| 1 | **Dual Output Grading** | Agent must produce BOTH a Markdown table AND a JSON KPI dict — scored independently |
| 2 | **Multi-Pass Refinement Loop** | `crop_region` -> `retry_region` -> `correct_cell` with reward for each CER improvement |
| 3 | **KPI Semantic Labeling** | Agent must identify which values are KPIs and assign meaningful keys (`revenue_fy2024`, `ebitda_margin`) |
| 4 | **Hallucination Penalty** | Agent penalized for inventing KPI keys not in ground truth (-0.05 per fake key) |
| 5 | **Noise Curriculum** | Three difficulty levels: clean -> noisy -> heavily degraded + rotated |

## Tasks (3 Levels of Difficulty)

### Task 1: Clean Printed Table (Easy)

- **Document:** Single sales report table (5 columns x 7 rows), clean white background
- **Generated by:** PIL ImageDraw with clean fonts and grid lines
- **Agent Goal:** Extract markdown table + 3 KPIs: `total_q1`, `total_q4`, `best_product`
- **Grading:** 50% cell-level markdown accuracy + 50% KPI exact match
- **Optimal steps:** 2 | **Max steps:** 5

### Task 2: Noisy Financial Statement (Medium)

- **Document:** Annual financial table with multi-level headers, Gaussian noise (intensity 18), 1.5 degree rotation
- **OCR Simulation:** Characters deliberately corrupted (`$` -> `S`, `%` -> `°/o`, `+` -> `÷`)
- **Agent Goal:** Extract markdown + 4 KPIs: `revenue_fy2024`, `ebitda_margin`, `yoy_growth_pct`, `net_income_fy2024`
- **Grading:** 40% markdown fidelity + 40% KPI match + 20% calibration confidence
- **Multi-pass:** Agent can crop and retry up to 3 column regions
- **Optimal steps:** 4 | **Max steps:** 10

### Task 3: Degraded Multi-Table Report (Hard)

- **Document:** TWO tables on one page (Operational KPIs + Financial Summary), heavy noise (intensity 35), 3.5 degree rotation, low contrast
- **Agent Goal:** Separate markdown for each table (divided by `---`) + nested KPI JSON `{"table1": {...}, "table2": {...}}`
- **Requires:** `switch_table` action to toggle between Table 1 and Table 2
- **Grading:** 35% per-table markdown + 35% KPI accuracy + 15% cross-table consistency + 15% step efficiency
- **Optimal steps:** 8 | **Max steps:** 15

## Action Space

| `action_type` | Required Fields | Description |
|---|---|---|
| `extract_table_md` | `markdown: str` | Submit a Markdown table extraction attempt |
| `extract_kpis` | `kpis: dict` | Submit a JSON dict of semantically labeled KPIs |
| `crop_region` | `region: {r1, r2}` | Zoom into a row range of the document text |
| `retry_region` | — | Re-extract the currently cropped region |
| `correct_cell` | `cell_row, cell_col, cell_value` | Fix a specific cell in the current markdown |
| `switch_table` | — | Toggle between Table 1 / Table 2 (Task 3 only) |
| `finalize` | — | Commit current outputs, end the episode, receive final score |

---

## Observation Space

| Field | Type | Description |
|---|---|---|
| `image_b64` | `str or null` | Base64 PNG of the document (sent on reset and after crop) |
| `text_hint` | `str` | Noisy simulated OCR text output (always available) |
| `reward` | `float` | Reward for the last action taken |
| `done` | `bool` | Whether the episode is complete |
| `cer` | `float or null` | Current Character Error Rate vs ground truth |
| `kpi_score` | `float or null` | Current KPI field accuracy (correct/total) |
| `error` | `str or null` | Error message if last action was invalid |
| `metadata` | `dict` | Contains step count, max_steps, active_table index |

---

## Reward Function (Incremental, Not Just Final)

Rewards are given at every step, not just on finalize:

| Signal | Reward Value |
|---|---|
| `extract_table_md` that improves CER | `+0.1 x (previous_CER - new_CER)` |
| `extract_kpis` with new correct field | `+0.10` per newly matched field |
| `extract_kpis` with hallucinated key | `-0.05` per invented key |
| `crop_region` + `retry_region` improves CER | `+0.05` |
| `correct_cell` that fixes an error | `+0.05` |
| `switch_table` (Task 3 navigation) | `0.00` (neutral) |
| Same action repeated 3x in a row (loop) | `-0.10` |
| `finalize` with no markdown or KPIs | `-0.20` |
| `finalize` with full submission | Final grader score `0.0 - 1.0` |
| Steps exceeding efficiency budget | `-0.02` per excess step |

---

## Grading Engine

All graders return values strictly in `[0.0, 1.0]`:

| Grader | Method |
|---|---|
| **CER** | Levenshtein edit distance / reference length, clamped to [0, 1] |
| **Markdown Score** | Parse both tables into rows, cell-by-cell exact match: `correct / total` |
| **KPI Score** | Per-field match — numeric values within 1% tolerance, strings exact |
| **Calibration** | Brier score on optional per-cell confidence: `1.0 - mean((conf - correct)^2)` |
| **Efficiency** | `max(0, 1.0 - 0.02 * max(0, steps_used - optimal_steps))` |

## Gradio UI (3 Tabs)

### Tab 1: RL Environment
Interactive demo of the graded tasks. Select a task, reset the episode, and step through actions. See live reward, CER, and KPI scores update in real-time.

### Tab 2: Upload Document
Upload any PDF or image (PNG, JPG, TIFF, BMP, WebP). **PyMuPDF** instantly extracts:
- All tables as Markdown (labeled by page and table number)
- Auto-detected KPIs as JSON (first-column labels + numeric values)
- Full page-by-page text
- Works on large multi-page PDFs

### Tab 3: Info
Task descriptions, API endpoint reference, and the full reward function table.

## Setup & Usage

### Local Development

```bash
pip install -r requirements.txt
python app.py
# Gradio UI at http://localhost:7860
# API at http://localhost:7860/health
```

### Docker

```bash
docker build -t ocr-table-rl .
docker run -p 7860:7860 ocr-table-rl
```

### Run Inference Baseline

```bash
# Start the environment server first, then:
export HF_TOKEN=your_huggingface_token
export API_BASE_URL=https://api-inference.huggingface.co/v1
export MODEL_NAME=Qwen/Qwen2.5-72B-Instruct
python inference.py
```

### Quick API Test

```bash
# Health check
curl https://SpandanM110-RL-OCR-Openenv.hf.space/health

# Reset to easy task
curl -X POST https://SpandanM110-RL-OCR-Openenv.hf.space/reset \
  -H "Content-Type: application/json" \
  -d '{"task": "clean_table"}'

# Submit markdown extraction
curl -X POST https://SpandanM110-RL-OCR-Openenv.hf.space/step \
  -H "Content-Type: application/json" \
  -d '{"action_type": "extract_table_md", "markdown": "| Product | Q1 |\n|---|---|\n| Widget A | 1200 |"}'

# Submit KPIs
curl -X POST https://SpandanM110-RL-OCR-Openenv.hf.space/step \
  -H "Content-Type: application/json" \
  -d '{"action_type": "extract_kpis", "kpis": {"total_q1": "10220", "best_product": "Widget E"}}'

# Finalize
curl -X POST https://SpandanM110-RL-OCR-Openenv.hf.space/step \
  -H "Content-Type: application/json" \
  -d '{"action_type": "finalize"}'
```

## Inference Output Format

The inference script emits strictly formatted stdout logs:

```
[START] task=clean_table env=ocr-table-rl model=Qwen/Qwen2.5-72B-Instruct
[STEP] step=1 action={"action_type":"extract_table_md",...} reward=0.05 done=false error=null
[STEP] step=2 action={"action_type":"extract_kpis",...} reward=0.30 done=false error=null
[STEP] step=3 action={"action_type":"finalize"} reward=0.82 done=true error=null
[END] success=true steps=3 rewards=0.05,0.30,0.82
```

Format rules:
- One `[START]` line at episode begin
- One `[STEP]` line per step, immediately after `env.step()` returns
- One `[END]` line after episode ends (always emitted, even on exception)
- `reward` and `rewards` formatted to 2 decimal places
- `done` and `success` are lowercase booleans: `true` or `false`
- `error` is the error string, or `null` if none

## Baseline Performance

| Task | Difficulty | Avg Score | Avg Steps | Notes |
|---|---|---|---|---|
| `clean_table` | Easy | ~0.75 | 2-3 | High accuracy on clean docs |
| `noisy_financial` | Medium | ~0.50 | 4-6 | Noise reduces first-pass accuracy |
| `degraded_report` | Hard | ~0.30 | 8-12 | Multi-table + heavy degradation |

Scores improve significantly with:
- Vision-capable models (GPT-4o, Claude Sonnet) using `image_b64`
- Multi-pass refinement (crop + retry + correct_cell)
- Better prompt engineering for KPI semantic labeling
